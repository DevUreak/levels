요구 기능 
- NFT 구매 함수 
- NFT 수익 인출 함수
- NFT 수익 인출 조회 함수 
- 구매 인출 테스트 구현

전체 요구 사항 
- tokenURI : 구매시 1씩 증가, 모든 NFT는 하나의 URI로 통합
- erc721 표준 사용 -> 오픈 제플린 
- 개발툴은 트러플, 하드햇 
- 예외처리와 이슈에대한 대응책은 최대한 처리 

생정자 데이터 처리 (수정 불가) 
- 최대 발행량 
- 마켓 오픈및 종료시간 
- NFT 민팅 가격
- 분배 지갑 : 수익 분배를 위한 지갑 리스트
- Token URI : 토큰 URI 

1. NFT 구매 함수 
-  함수이름 : buyNFT 
-  기능 : ETHER를 지불하면 새로운 NFT 발행 
-  요구사항 : 
    1. ether 전송시 call사용 
    2. 구매완료시 tokenid 1 증가 최초구매자는 1 
    3. 최대 발행량 초과 불가 
    4. NFT가격 이상으로 보내면 남은 금액은 페이백 
    5. 마켓 구매 시간 범위 내에서만 구매가능 그외는 불가능 


2. 인출 함수 
- 함수 이름 : withdraw
- 기능 : 자신의 수익만 인출이 가능함 
- 요구사항 : 
    1. ether 전송시 call사용 
    2. 컨트랙트 생성시 등록된 지갑에있는 주소만 인출이 가능 + 함수를 호출한 본인껏만 인출이댐 
    3. 지갑 배열 등록한것만큼 n/1해서 수익분배 

3. 인출 조회 함수 
- 함수 이름 : getWithdraw
- 기능 : 자신이 현재 인출가능한 수익 보여줌 
- 요구사항 : 
    1. 자신의 지갑 정보만 조회 가능
